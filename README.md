# House Price Prediction

Predicts house prices from property features (area, bedrooms, bathrooms,
location, age, parking, etc.) using scikit-learn, with a Streamlit app for
making predictions interactively.

**Live demo:** https://sam-house-price-prediction.streamlit.app

## Dataset

The data (`data/housing_data.csv`, ~1,500 rows) is generated by
`src/generate_dataset.py` so the project runs without downloading anything.
Prices are in Indian rupees at metro-city rates (roughly ₹3,000–13,000 per
sq ft depending on locality), and the data intentionally includes missing
values, duplicates, and outliers so the cleaning step has real work to do.
The same pipeline works on real datasets (Kaggle Housing Prices, Ames,
California Housing) with minor column-name changes in `src/data_preprocessing.py`.

| Column | Description |
|---|---|
| `area` | Living area in square feet |
| `bedrooms`, `bathrooms`, `stories` | Room / floor counts |
| `location` | City Centre, Prime Suburb, Suburb, Outskirts, Premium Township |
| `house_age` | Age in years |
| `parking` | Parking spots |
| `main_road` | Faces a main road (yes/no) |
| `furnishing_status` | furnished / semi-furnished / unfurnished |
| `price` | Target — sale price in ₹ |

## Setup

```bash
git clone https://github.com/SamGabriel-Here/house-price-prediction.git
cd house-price-prediction

python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Running the pipeline

```bash
python src/generate_dataset.py     # create data/housing_data.csv
python src/eda.py                  # save charts to reports/figures/
python src/data_preprocessing.py   # clean -> data/housing_clean.csv
python src/train_model.py          # train + compare -> models/house_price_model.pkl
streamlit run app/app.py           # web app
```

## How it works

**EDA** (`src/eda.py`) saves five charts to `reports/figures/`: price
distribution, area vs price by location, bedrooms vs price, a correlation
heatmap, and average price by location. Area correlates most strongly with
price; house age is negatively correlated; City Centre and Premium Township
are the most expensive locations.

**Preprocessing** (`src/data_preprocessing.py`):
- missing values → median (numeric) / mode (categorical)
- duplicate rows dropped
- `price` and `area` outliers capped with the IQR rule
- engineered features: `total_rooms` (bedrooms + bathrooms), `is_new` (age ≤ 5)
- one-hot encoding for categoricals, `StandardScaler` for numerics, both
  inside a scikit-learn `Pipeline` so the saved model preprocesses new
  inputs the same way it was trained

**Training** (`src/train_model.py`) compares three models on an 80/20 split
and saves whichever scores highest on the test set:

| Model | MAE (₹) | RMSE (₹) | R² |
|---|---:|---:|---:|
| XGBoost | 10.9 Lakh | 18.4 Lakh | 0.889 |
| Random Forest | 11.5 Lakh | 18.8 Lakh | 0.884 |
| Linear Regression | 14.2 Lakh | 20.9 Lakh | 0.856 |

The tree models win here because location changes the price *per square
foot* — an interaction a plain linear model can't fully capture. Exact
numbers vary slightly by library version.

**App** (`app/app.py`) — a Streamlit form for the house features that returns
a price estimate from the saved model.
